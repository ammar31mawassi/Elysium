# Repository Working Rules

## User-Provided Assets Take Precedence

- When Ammar provides an image, logo, file, document, dataset, design, or other project asset, use that exact supplied asset for the requested purpose.
- Do not generate, redraw, replace, reinterpret, or substitute a provided asset without Ammar's explicit approval.
- If an alternative may improve the product, explain the alternative first and wait for confirmation before creating or using it.
- Preserve original supplied files. Copy them into application-owned folders only when needed for builds or deployment.

## Deploy Completed Implementations

- After an implementation passes the appropriate local validation, deploy the applicable changes to the live Base44 application unless Ammar explicitly says not to deploy that work.
- Deploy frontend changes with a production build followed by the Base44 site deployment workflow.
- When an implementation changes Base44 entity schemas or Agent definitions, review the full synchronization impact and push the corresponding entities or Agents as part of that deployment.
- Do not deploy incomplete, failing, or unverified work.
